---

## ΦΑΣΗ 1: ΑΜΕΣΗ ΔΡΑΣΗ (Λεπτά 0-15)

*   [ ] **Επιβεβαίωση Alert:** Επιβεβαίωσε τη λήψη του alert. Κατέγραψε την ακριβή ώρα: `__:__ UTC`
*   [ ] **Δημιουργία Καναλιού Επικοινωνίας:** Δημιούργησε ασφαλές κανάλι επικοινωνίας (π.χ. dedicated Slack channel, conference call). Όνομα καναλιού: `incident-YYYYMMDD-XXXX`
*   [ ] **Σύγκληση Ομάδας Άμεσης Αντίδρασης (Core IRT):** Κάλεσε ΑΜΕΣΑ τους παρακάτω στο κανάλι επικοινωνίας:
    *   [ ] CISO
    *   [ ] Head of Operations
    *   [ ] DPO
    *   [ ] Technical Lead (π.χ. Crypto Officer)
*   [ ] **Έναρξη Ημερολογίου Περιστατικού:** Άνοιξε νέο, ασφαλές έγγραφο για την καταγραφή ΟΛΩΝ των ενεργειών. `incident-log-YYYYMMDD-XXXX.md`

---

## ΦΑΣΗ 2: ΑΞΙΟΛΟΓΗΣΗ & ΤΑΞΙΝΟΜΗΣΗ (Λεπτά 15-45)

*   [ ] **Συλλογή Πληροφοριών (5 Ws):** Συγκέντρωσε τις πρώτες, κρίσιμες πληροφορίες.
    *   **What:** Τι συστήματα/υπηρεσίες επηρεάζονται; (π.χ. Issuing CA, OCSP, Timestamping)
    *   **Who:** Ποιος ανίχνευσε το περιστατικό;
    *   **When:** Πότε ξεκίνησε το περιστατικό;
    *   **Where:** Πού παρατηρείται η ανωμαλία; (π.χ. DC1, DC2, Cloud)
    *   **Why:** Ποια είναι η αρχική υποψία για την αιτία; (π.χ. DDoS, malware, insider)

*   [ ] **Αξιολόγηση Αντικτύπου (Βάσει Παραρτήματος 3 ΕΕΤΤ):** Αξιολόγησε τον αντίκτυπο σε κάθε κατηγορία.

| Κριτήριο | Εκτίμηση Αντικτύπου | Επίπεδο (1-5) | Σημειώσεις |
|---|---|---|---|
| **1. Αριθμός χρηστών** | `(π.χ. <100, 100-1000, >1000)` | | |
| **2. Διάρκεια** | `(π.χ. <1h, 1-6h, >6h)` | | |
| **3. Γεωγραφική εξάπλωση** | `(π.χ. Εντός Οργανισμού, GR, EU)` | | |
| **4. Επίπτωση στα δεδομένα** | `(π.χ. Καμία, Απώλεια Ακεραιότητας, Απώλεια Εμπιστευτικότητας)` | | `ΚΡΙΣΙΜΟ: Αφορά προσωπικά δεδομένα;` |
| **5. Επίπτωση στην υπηρεσία** | `(π.χ. Καμία, Διακοπή, Παραποίηση)` | | `ΚΡΙΣΙΜΟ: Επηρεάζεται η δημιουργία/επικύρωση υπογραφών;` |
| **6. Οικονομικός/Κοινωνικός αντίκτυπος** | `(π.χ. Μικρός, Σημαντικός, Σοβαρός)` | | |

*   [ ] **Οριστική Ταξινόμηση:** Βάσει του παραπάνω πίνακα, κύκλωσε την τελική ταξινόμηση:

    **ΕΠΙΠΕΔΟ 1 -- ΕΠΙΠΕΔΟ 2 -- ΕΠΙΠΕΔΟ 3 -- ΕΠΙΠΕΔΟ 4 -- ΕΠΙΠΕΔΟ 5**

*   [ ] **Καταγραφή Αιτιολόγησης:** Στο ημερολόγιο του περιστατικού, τεκμηρίωσε σύντομα γιατί επελέγη αυτό το επίπεδο.

---

## ΦΑΣΗ 3: ΑΠΟΦΑΣΗ & ΕΝΕΡΓΟΠΟΙΗΣΗ ΚΟΙΝΟΠΟΙΗΣΗΣ (Λεπτά 45-60)

*   [ ] **Λήψη Απόφασης:** Βάσει της ταξινόμησης, επίλεξε την κατάλληλη ενέργεια.

    *   **Εάν το Επίπεδο είναι 1 ή 2:**
        *   [ ] Κήρυξε το ως **"Επιχειρησιακό Περιστατικό"**.
        *   [ ] Συνέχισε με τις εσωτερικές διαδικασίες περιορισμού και ανάκαμψης.
        *   [ ] Επιβεβαίωσε ότι **ΔΕΝ απαιτείται** άμεση αναφορά στην ΕΕΤΤ.

    *   **Εάν το Επίπεδο είναι 3, 4 ή 5:**
        *   [ ] Κήρυξε το ως **"Σημαντικό Περιστατικό Ασφαλείας που Απαιτεί Αναφορά"**.
        *   [ ] **ΚΡΙΣΙΜΗ ΕΝΕΡΓΕΙΑ:** Δώσε εντολή στον **EETT Liaison (DPO)** να ξεκινήσει **ΑΜΕΣΑ** τη σύνταξη της "Αρχικής Αναφοράς Περιστατικού" χρησιμοποιώντας το πρότυπο `07b_Incident_Forms_GR.xlsx`.
        *   [ ] **ΕΠΙΣΗΜΑΝΣΗ:** Επιβεβαίωσε και κατέγραψε στο ημερολόγιο την **ακριβή ώρα "γνώσης του περιστατικού" (time of becoming aware)**. Η προθεσμία των **24 ωρών** για την υποβολή στην ΕΕΤΤ ξεκινά από αυτή τη χρονική στιγμή.
            *   **Ώρα Γνώσης:** `__:__ UTC`
            *   **Προθεσμία Υποβολής στην ΕΕΤΤ:** `[Ώρα Γνώσης + 24 ώρες]`
